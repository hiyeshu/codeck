核心论点

AI 让个人开发者成为 10x 团队

不是用 AI 替代人，而是用 AI 放大人的判断力。一个人 + Claude Code 可以做到过去 5 个人做的事。

为什么选择三层架构

  • 内容与视觉分离
    deck.json 只管内容，design.json 只管视觉。改颜色不碰文字，改文字不碰排版。
  • 确定性渲染
    同样的 JSON 输入，永远产出同样的 HTML。不再靠 AI 碰运气。
  • 可组合
    18 种 block type 自由排列，像乐高一样搭建任意页面结构。
  • 可测试
    每种组件有快照测试，改了 registry 立刻知道哪里坏了。
  • Step 1: Claude 生成 spec
    纯 JSON 结构，不含任何视觉信息
  • Step 2: Registry 渲染默认 HTML
    确定性函数，白底系统字体
  • Step 3: Design Compiler 应用样式
    注入 CSS 变量、字体、背景效果
compiler/registry/blocks/statement-hero.ts
export function renderStatementHero(
  id: string,
  props: Record<string, unknown>
): string {
  const label = props.label
    ? `<p class="label">${escape(props.label)}</p>`
    : '';
  const title = `<h1>${escape(props.title)}</h1>`;
  const body = props.body
    ? `<p class="body">${escape(props.body)}</p>`
    : '';
  return wrapBlock(id, 'statement-hero',
    `<div class="statement-hero">${label}${title}${body}</div>`
  );
}

设计原则

每个 block type 都有一个确定性的 HTML 模板。同样的输入永远产出同样的输出。

注意

design compiler 不修改 DOM 结构，只注入 CSS。如果需要结构变更，改 spec 重新渲染。

已验证

同一份 spec 渲染两次，diff 为空。确定性保证通过。

破坏性变更

deck.json 格式从 pages+blocks 改为 flat element tree。旧格式需要通过 migrate 命令转换。
